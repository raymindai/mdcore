## 1. 전략 전환 요약

### Before (v4.0)

> "AI와 사람 사이의 콘텐츠 인프라를 소유한다."

- 포지셔닝: Markdown 인프라 엔진 회사
- 타겟: 개발자
- 제품 = 엔진. mdfy.cc는 쇼케이스.
- 원칙: "에디터를 만들지 마라. 엔진을 만들어라."

### After (v5.0)

> **"The fastest way from thought to shared document."**

- 포지셔닝: AI 출력물의 크로스플랫폼 퍼블리싱 도구
- 타겟: AI를 사용하는 모든 지식 노동자 (개발자 → 일반 사용자로 확장)
- 제품 = mdfy.cc 자체가 핵심 사업. 엔진은 내부 기술.
- 원칙: "Markdown은 엔진이지 인터페이스가 아니다."

### 왜 바꿨는가

1. **제품 5개 해자 0개 → 제품 1개 해자 1개가 낫다.** 1인 팀이 5개 표면에 분산하면 모두 중간만 한다.
2. **"Markdown 인프라"는 개발자에게만 의미 있다.** 사용자 풀이 너무 좁다.
3. **리서치 결과, "AI 출력 → 퍼블리셔블 문서" 포지션이 비어있다.** 펀딩 받은 경쟁자 0개. ShareGPT는 deprecated.
4. **AI 회사가 구조적으로 복제 불가능한 포지션이다.** OpenAI는 Claude 출력을 위한 도구를 안 만들고, Anthropic도 ChatGPT 출력을 위한 도구를 안 만든다.

## 2. 핵심 해자 (4개)

### 해자 1: 바이럴 뱃지 — "Published with mdfy.cc"

모든 무료 문서 하단에 바이럴 뱃지를 표시한다. 공유되는 문서 자체가 광고판이 된다.

검증된 패턴:
- **Carrd** (1인): "Made with Carrd" 뱃지로 마케팅 비용 $0, ARR $1.5-2M
- **Beehiiv**: 이메일 푸터만으로 월 $10M 상당의 무료 노출
- **Buttondown** (1인): "Powered by" 푸터로 꾸준한 5-10% 월간 성장

### 해자 2: 크로스 AI 레이어

```
ChatGPT  ──┐
Claude   ──┼──► mdfy.cc ──► 프로페셔널 문서 URL
Gemini   ──┤
Perplexity─┘

이 레이어는 AI 회사가 절대 만들지 않는다.
(경쟁사 출력을 모아주는 도구를 왜 만들겠나?)
```

### 해자 3: 렌더링 품질

코드 하이라이팅 + KaTeX 수학 + Mermaid 다이어그램을 하나의 URL에서 완벽하게 지원하는 무료 도구가 없다. mdcore 엔진(Rust/WASM)이 이 품질을 보장한다.

### 해자 4: 공유 = 배포

```
사용자 A가 문서 공유 → 수신자 B가 mdfy.cc 첫 방문
  → B: "나도 이렇게 공유하고 싶다" → B가 사용자가 됨
    → B가 문서 공유 → 수신자 C, D, E...
```

Dropbox가 "공유 폴더"로, Notion이 "공유 페이지"로 성장한 것과 같은 구조.

### 제품-브랜드 매핑

| 브랜드 | 역할 | 사용자가 보는 것 |
|---|---|---|
| **mdfy.cc** | 제품 (서비스) | "여기서 문서 만들고 공유한다" |
| **mditor** | 에디터 경험 (Writer/Source 모드) | "쓰는 느낌" |
| **mdcore** | 엔진 (보이지 않음) | 개발자만 안다 (npm, API) |
| **Chrome ext** | 캡처 진입점 | "AI 답변 → 원클릭 퍼블리시" |

### 컴파일 타겟 매트릭스

```
mdcore 엔진 (Rust/comrak)
  │
  ├── wasm32-unknown-unknown  → 브라우저 (mdfy.cc, Chrome ext)
  ├── aarch64-apple-darwin    → macOS QuickLook 플러그인
  ├── x86_64-unknown-linux    → API 서버
  └── npm 패키지              → @mdcore/engine
```

## 4. 사용자 여정

### 킬러 시나리오

1. **AI 사용자**: ChatGPT에서 답변 → Chrome 익스텐션 클릭 → mdfy.cc에서 편집 → URL 공유 → 끝
2. **기획자**: mdfy.cc에서 문서 작성 → 링크 공유 → 팀 리뷰 (앱 불필요)
3. **개발자**: Source 모드로 기술 문서 작성 → 코드+수식+다이어그램 포함 → embed

### Web-Native Document의 핵심

**"문서가 파일이 아니라 URL이다."**

| | 올드 패러다임 | Web-Native (mdfy) |
|---|---|---|
| 문서 = | `.docx` 파일 | URL |
| 보려면 | 앱 설치 / 로그인 | 브라우저만 |
| 공유 = | 파일 첨부 | 링크 전송 |
| 포맷 | 인쇄 기준 (A4) | 스크린 기준 (반응형) |
| 생성자 | 사람만 | 사람 + AI + API |

## 5. 수익 모델

### 가격 구조 (시장 검증 기반)

| 티어 | 가격 | 핵심 |
|------|------|------|
| **Free** | $0 | 무제한 문서, 7일 만료, 바이럴 뱃지 |
| **Pro** | $8/mo | 만료 없음, 커스텀 도메인, 조회 분석, 워터마크 제거, 비밀번호 보호 |
| **API** | $0.01/doc | 프로그래매틱 문서 생성 |

> $8/mo는 Notion 커스텀 도메인($8/mo), Obsidian Publish($8/mo)와 동일한 시장 검증 가격대.

### 왜 이 모델이 작동하는가

- **무료 = 바이럴 엔진**: 7일 만료 + 뱃지로 매일 마케팅이 돌아감
- **Pro 전환 트리거**: 만료 제거 + 커스텀 도메인은 검증된 유료화 요소
- **업계 전환율**: 무료→유료 2-5% (보수적 3% 적용 시)
- **TAM**: ChatGPT WAU 2억명의 5% = 1,000만 잠재 사용자

## 6. 실행 로드맵

### 우선순위 원칙

**바이럴 루프를 먼저 돌린다.** 나머지는 루프가 도는 상태에서 확장한다.

### Phase 순서

```
Phase 1 (즉시)          Phase 2 (4주 후)       Phase 3 (8주 후)
────────────────────   ──────────────────────  ──────────────────
바이럴 뱃지 추가        Pro 결제 (Stripe)       API 오픈
7일 만료 (무료)         커스텀 도메인            팀 워크스페이스
Chrome 익스텐션 완성    조회 분석
AI 감지 + 자동 정리     워터마크 제거
바이럴 루프 시작
```

### 확장 제품 (바이럴 루프 확인 후)

| 제품 | 역할 | 시점 |
|---|---|---|
| **npm 패키지** (@mdcore/engine) | 개발자 생태계 = 해자 강화 | 바이럴 확인 후 |
| **WYSIWYG** (mditor Writer 모드) | 사용자 풀 확장 (비개발자) | 매출 확인 후 |
| **QuickLook 플러그인** | macOS 브랜드 터치포인트 | 여유 있을 때 |
| **CLI** (@mdcore/terminal) | 개발자 도구 생태계 | 여유 있을 때 |
| **API 플랫폼** (mdcore.ai) | B2B 매출 | 수요 발생 시 |

> 모든 확장 제품은 계획에 포함되어 있다. 다만 **바이럴 뱃지 + Chrome 익스텐션이 먼저**다.

## 7. 경쟁 환경 (리서치 결과)

### AI 출력 공유 — 경쟁자 부재 확인

| AI 플랫폼 | 현재 공유 기능 | 한계 |
|---|---|---|
| ChatGPT | 대화 스냅샷 링크 | 대화 UI 그대로 노출, 문서가 아님 |
| Claude | 공유 기능 없음 | 팀 내부만 가능 |
| Gemini | Google Docs 연동 | 독립 공유 URL 불가 |
| Perplexity | Pages (가장 가까움) | 검색 결과 전용, 범용 아님 |

**"AI 출력 → 깔끔한 퍼블리셔블 URL" 전용 도구 = 펀딩 받은 스타트업 0개.**

### 인접 경쟁자

| 플레이어 | 위치 | mdfy가 이기는 포인트 |
|---|---|---|
| HackMD ($5/mo) | 협업 MD 에디터 | 퍼블리싱이 아니라 협업 도구 |
| Notion ($10/mo) | 올인원 워크스페이스 | 무겁고 로그인 필요 |
| Rentry.co (무료) | 기본 MD 페이스트빈 | 렌더링 품질 10x 차이 |
| Unmarkdown ($8/mo) | MD→예쁜 문서 | 로그인 필수, AI 특화 아님 |

### 검증된 레퍼런스

- **Mintlify** (YC W22, $21M 시리즈A from a16z): "Markdown을 아름답게 퍼블리시"로 검증. 단, API 문서 특화. AI 출력 공유는 비어있음.
- **Excalidraw**: 오픈소스 + 103K GitHub stars + 임베더블 → 생태계 해자 모델.

## 9. 핵심 원칙 (업데이트)

1. ~~에디터를 만들지 마라~~ → **Markdown은 엔진이지 인터페이스가 아니다.** 사용자는 MD를 몰라도 된다.
2. **바이럴 루프가 먼저.** 뱃지 + 공유 = 무료 마케팅. 이것 없이 다른 것 만들지 않는다.
3. **크로스 AI = 구조적 해자.** AI 회사가 절대 복제 못하는 포지션.
4. **제품 1개, 해자 1개.** 여러 제품을 동시에 런칭하지 않는다. 확장은 순차적으로.
5. **Web-Native Document.** 문서는 파일이 아니라 URL이다.
6. **Zero friction.** 로그인 없이, 설명 없이, 3초 안에 가치 전달.
7. **$8/mo.** 시장 검증된 가격. 이것보다 비싸면 망설이고, 싸면 가치를 떨어뜨린다.
