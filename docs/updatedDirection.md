## 0. v5.0에서 뭘 틀렸는가

v5.0에서 "제품 1개, 해자 1개"라고 했다. 실제로는:
- mdfy.app 웹앱 기능 추가 (WYSIWYG, 자동저장, My Documents)
- VS Code 익스텐션 (WYSIWYG + 클라우드 싱크 + 이미지 업로드)
- Chrome 익스텐션 (대화 캡처)
- @mdcore/engine npm 패키지 추출
- mdcore-ai 랜딩 페이지

5개를 동시에 만들었다. 원칙을 어겼다.

**결과**: 바이럴 뱃지 아직 없음. 7일 만료 없음. Stripe 없음. 수익 $0.
핵심 해자를 안 만들고 표면만 넓힌 상태.

## 1. 전략 v6.0: 순차 집중

### 원칙

**하나를 끝내고 다음으로 간다.** 동시에 2개 이상 진행하지 않는다.

"끝낸다"의 정의: 배포 완료 + 사용자가 쓸 수 있는 상태 + 측정 가능한 지표 확인.

### 포지셔닝 (v5.0 유지)

> **"The fastest way from thought to shared document."**

- mdfy.app가 핵심 제품.
- mdcore는 내부 기술이자 미래 에코시스템의 기반.
- 사용자에게 보이는 건 mdfy, 개발자에게 보이는 건 mdcore.

## 2. 실행 순서

### Step 1: 한 번에 런칭 ← 지금

모든 표면을 동시에 런칭한다. 기능은 이미 만들어져 있다.

| 런칭 항목 | 상태 | 할 일 |
|----------|------|-------|
| mdfy.app 웹앱 | ✅ 라이브 | 바이럴 뱃지 추가 |
| Chrome 익스텐션 | ✅ 코드 완료 | Chrome Web Store 출시 |
| VS Code 익스텐션 | ✅ 코드 완료 | Marketplace 출시 |
| 런칭 포스팅 | 📝 초안 있음 | HN, Reddit, Twitter, Product Hunt |

**전부 무료.** 만료 없음. 제한 없음. AI 피쳐 포함 모든 기능 공짜.
**유료화 나중.** 사용자 수가 늘고 안정화된 후 시작한다.

**완료 조건**: 3개 표면 동시 라이브 + 런칭 포스팅 게시.

---

### Step 2: 바이럴 루프 관찰 + 제품 개선

런칭 후 사용자 반응을 보고 제품을 개선한다.

| 할 일 | 이유 |
|-------|------|
| 바이럴 루프 측정 | 뱃지 → 유입 → 문서 생성 → 공유 흐름 추적 |
| 사용자 피드백 반영 | 실사용에서 나오는 문제 해결 |
| AI 감지 + 정리 강화 | 크로스 AI 해자 강화 |

---

### Step 3: 유료화

사용자 수가 충분히 늘고 안정화된 후. 뭘 유료로 할지는 사용자 데이터를 보고 결정.

| 후보 | 판단 기준 |
|------|----------|
| Pro 티어 | 어떤 기능에 파워 유저가 몰리는지 |
| 커스텀 도메인 | 요청이 얼마나 오는지 |
| API 종량제 | 프로그래매틱 사용 패턴 |

---

### Step 4: 에코시스템 확장

순서는 수요에 따라 결정:
- @mdcore/engine npm 공개
- mdfy for Mac (오프라인 + 싱크)
- QuickLook 플러그인
- @mdcore/terminal CLI
- mdcore.ai API

## 4. 핵심 해자 (v5.0 유지)

1. **바이럴 뱃지** — 공유 = 마케팅 (Carrd, Beehiiv 검증 패턴)
2. **크로스 AI 레이어** — AI 회사가 구조적으로 복제 불가
3. **렌더링 품질** — Rust WASM (comrak), KaTeX + Mermaid + 하이라이팅 통합
4. **공유 = 배포** — 문서를 보면 만들고 싶어지는 바이럴 구조

---

## 5. 수익 모델

**런칭 시 전부 무료.** AI 피쳐 포함 모든 기능을 제한 없이 제공한다.
사용자 수가 충분히 늘고 서비스가 안정화되면 유료화를 시작한다.

| 단계 | 모델 |
|------|------|
| **런칭** | 전 기능 무료. AI 피쳐, 무제한 문서, 영구 보존, 바이럴 뱃지 |
| **안정화 후** | Free 유지 + Pro 티어 도입 (가격/기능은 사용자 데이터 보고 결정) |

## 6. 비전: MDcore 에코시스템

Step 5 이후의 풀 비전. **지금 만들지 않는다.** 하지만 이 방향을 알고 있으므로, 지금 만드는 코드가 이 구조를 방해하지 않도록 한다.

```
사용자 표면 (같은 엔진, 같은 품질, 같은 경험)
─────────────────────────────────────────────
mdfy.app      웹 — 핵심 제품
Chrome ext   브라우저 — AI 대화 캡처
VS Code ext  에디터 — 오프라인 편집 + 싱크
mdfy for Mac 데스크탑 — 네이티브 오프라인
QuickLook    macOS — 스페이스바 프리뷰
CLI          터미널 — 개발자 도구
API          서버 — 프로그래매틱 접근
─────────────────────────────────────────────

공유 엔진 (@mdcore/engine)
─────────────────────────────────────────────
Rust WASM    파싱 + 렌더링 (comrak)
JS 후처리    하이라이팅 + 수식 + 다이어그램
변환         HTML↔MD, AI 대화, CLI, CSV, LaTeX...
─────────────────────────────────────────────

클라우드 (Supabase)
─────────────────────────────────────────────
문서 저장 · 사용자 인증 · 실시간 싱크 · 버전 관리
─────────────────────────────────────────────
```

**온라인 ↔ 오프라인**: 모든 표면에서 만든 문서가 클라우드를 통해 동기화.
**UX 최고 수준**: 각 표면이 해당 플랫폼 네이티브 경험의 최상위.

## 7. 원칙 (v6.0)

1. **먼저 쓰게 하고, 나중에 받는다.** 제한 없이 무료로 제공. 유료화는 사용자가 모인 후.
2. **한 번에 런칭, 피드백으로 개선.** 완벽하게 만들고 내놓지 않는다. 내놓고 고친다.
3. **Markdown은 엔진이지 인터페이스가 아니다.** 사용자는 MD를 몰라도 된다.
4. **크로스 AI = 구조적 해자.** AI 회사가 절대 복제 못하는 포지션.
5. **Web-Native Document.** 문서는 파일이 아니라 URL이다.
6. **Zero friction.** 로그인 없이, 3초 안에 가치 전달.
7. **모든 표면, 같은 경험.** 웹, VS Code, Chrome, Mac — 어디서든 동일한 품질.
